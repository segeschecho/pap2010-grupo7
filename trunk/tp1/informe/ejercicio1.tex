\chapter{Problema 1}

\section{Edit Step Ladders}

\subsection{Enunciado}

Un $edit$ $step$ es una transformación de una palabra $x$ en otra palabra $y$, tal que $x$ e $y$ son palabras en un diccionario de palabras y además $x$ puede ser transformada en $y$ $agregando$, $borrando$ o $cambiando$ una letra. La transformación de $dig$ a $dog$ o de $dog$ a $do$ son ambas $edit$ $steps$.

Un $edit$ $step$ $ladder$ es una secuencia ordenada alfabéticamente $w_1$, $w_2$ ... $w_n$ tal que la transformación de $w_i$ a $w_{i+1}$ es un $edit$ $step$ para todos los $i$ de 1 a n-1.

Se pide entonces, dado un diccionario de palabras, calcular el $edit$ $step$ $ladder$ mas largo.

\textbf{Input:}

El input consiste en un diccionario de palabras (en minúscula y ordenada alfabéticamente) una por linea. Ninguna palabra tiene más de 16 letras y no hay mas de 25000 palabras en el diccionario.

\textbf{Output:}

El output consiste en un numero, que indica el $edit$ $step$ $ladder$ mas largo.

\subsection{Solución}
Para llegar a la solución de este problema, se pensaron varios modelos y algoritmos. Algunos eran deficientes en cuanto a complejidad, tardando demasiado tiempo en resolver el problema y otros no lo resolvían.

El algoritmo final, resuelve el problema en un tiempo razonable y por ende su complejidad es bastante buena. El mismo responde a un modelo iterativo, que depende de las palabras de entrada del problema y que va reutilizando datos que fueron calculados en pasos anteriores. Este método utiliza el concepto de que cada información calculada anteriormente es óptima, y por lo tanto para llegar a la solución óptima del valor actual, la misma se obtiene utilizando las soluciones optimas anteriores. Este concepto es muy importante y es el que define al método de programación dinámica, utilizado en este problema.

Comenzando entonces a describir el algoritmo creado para resolver el máximo edit step ladder de una lista de palabras, podemos decir que el mismo consta de cuatro grandes pasos:

\begin{enumerate}

\item Paso 1: es en donde se recorre cada una de las palabras desde el principio hasta el final.
\item Paso 2: se encarga de, dado una palabra actual ($P$), generar otras palabras que se consideren edit step de dicha $P$, es decir, que difieran de ésta en a lo sumo un caracter (ya sea quitándolo, agregándolo, o modificándolo).
\item Paso 3: utiliza las palabras generadas en el paso anterior y su función es verificar si cada una de éstas, está presente antes de $P$. Si alguna de éstas palabras es hallada, quiere decir que existe un edit step, por lo tanto, se puede agrandar en uno la distancia del step ladder de dicha palabra. En este paso, se utiliza lo calculado en pasos anteriores, es decir que cuando una palabra es hallada, la misma tiene asociado un valor que corresponde al mayor edit step ladder que termina en esa palabra. Por esta razón se ahorran cálculos innecesarios, que además ya fueron calculados anteriormente. Cabe aclarar que por cada palabra que es hallada, se verifica si se puede maximizar el edit step ladder hasta $P$, dicho de otra forma, se toma la palabra hallada que tenga mayor edit step ladder, de forma tal, que el valor asociado a la palabra actual $P$, sea el óptimo. En este paso entra en juego el principio de optimalidad que se plantea en la programación dinámica, ya que para asignarle el mejor step ladder que termina en la palabra actual $P$ (es decir el de mayor largo), el algoritmo se basa en decidir, cual es la palabra que maximiza su escalera, donde cada una de estas palabras tiene asociado con sigo, el mayor edit step ladder hasta ella (un valor óptimo). Por esta razón, podemos decir que siempre que se tome el mayor valor dentro de las palabras que pueden ir de ellas a $P$, el valor de la palabra actual $P$ será óptimo.
\item Paso 4: consiste en verificar si el edit step ladder calculado para $P$, es el mas grande entre todos los que se calcularon anteriormente. Si esto es cierto, el valor de edit step ladder de $P$ pasa a ser el posible valor que se retornará una vez que se finalice el recorrido de todas las palabras.
\end{enumerate}

Para dejar un poco más en claro, el funcionamiento del algoritmo de solución, a continuación se muestra la ecuación de recurrencia asociada(donde $N$ es la cantidad de palabras recibidas e $i$ es el indice que identifica a una palabra dentro de la lista):

$maxStepLadders( N ) = \left\{ \begin{array}{lll}
0 & {\rm si \ } N = 0 \\
1 & {\rm si \ } N = 1 \\
MAX_{1<= i <= N}( maxStepLocal( i ) ) & {\rm si \ } N > 1
\end{array}\right.
$

$maxStepLocal(i) = \left\{ \begin{array}{ll}
1 + MAX_{j \in posiblesStep(N)}(maxStepLocal(j)) & {\rm si \ } \exists(j, palabras [j] \in PosiblesStep(i)) \\
1 & {\rm sino \ }
\end{array}\right.
$

Como se puede observar, $maxStepLadders$ devuelve el resultado buscado, y en ella se puede ver de forma sobresaliente, que se efectúan los pasos $uno$ (recorrer las palabras) y $cuatro$ (quedarse con el mejor edit step ladder). En maxStepLocal, podemos observar que se llevan a cabo los pasos $dos$ (generar los posibles steps para una palabra) y $tres$ (seleccionar la que maximice el edit step ladder a la palabra actual).


\subsection{Por que esta solución ?}

En esta sección explicaremos por que este procedimiento funciona y llega a calcular la solución buscada. Como ya hemos visto, la solución consta de 4 grandes pasos que hemos mencionado anteriormente y que en esta sección se irán nombrando.

Primero veremos que dicha solución es un algoritmo y termina. Es fácil de ver si se observa el paso 1, donde en él se van recorriendo todas las palabras una por una, desde el principio hasta el final. Lo que demuestra que el algoritmo en algún momento termina, mas específicamente, cuando se llega al final de la lista de palabras que viene dada en la entrada del problema. Notamos también que las operaciones que se realizan dentro de este ciclo, terminan en algún momento, para esto tomaremos en cuenta los pasos restantes. El paso 2, genera los posibles step para la palabra actual $P$, este procedimiento esta acotado primeramente por el largo de $P$ y también por la cantidad de letras en el alfabeto (para el caso de inserción y modificación), por esta razón, podemos observar que la cantidad de operaciones a realizar esta acotada por alguna de estas dos condiciones. El paso 3, busca cada palabra generada desde el inicio hasta $P$, esto se hace buscando binariamente aprovechando que las palabras se encuentran ordenadas alfabéticamente. Como se puede observar este algoritmo también termina y como la cantidad de palabras a buscar es finita, el paso 3 finaliza en algún momento. El paso 4 finaliza trivialmente, ya que se verifica si el valor de edit step ladder calculado es mas grande que el mejor de los anteriores que se tiene guardado.

Luego de argumentar que la solución es un algoritmo y termina, resta argumentar por que la solución funciona y encuentra el valor buscado dentro de la lista de palabras dada en la entrada del problema. Esto lo podemos ver tomando un ejemplo como referencia y explicando por que en cada ciclo, en donde se toma una palabra (que se convierte en la palabra actual $P$)y se calcula el resultado al edit step mas largo que termina en $P$, puede ser el posible resultado del problema si se toma el valor mas grande entre el mayor de los edit steps calculados anteriormente y el del ciclo actual.

En este ejemplo vemos como el algoritmo llega al valor buscado. Pero podemos argumentar este funcionamiento correcto de una forma un poco mas formal. Para esto entonces, supongamos que nuestro algoritmo no funciona correctamente. Sea $L$ una lista de palabras ordenadas alfabéticamente $C_1, C_2,...,C_k$, y sea la lista $Lmax$ que es sublista de $L$ y que genera el edit step ladder máximo de valor $editStepLadderMax$, donde $Lmax$ $=$ $C_{i1}, C_{i2}, ... , C_{ij}$ (con $1 \le i1 < i2 < ... < ij \le k$). Como nuestro algoritmo no calcula correctamente el valor $editStepLadderMax$, quiere decir que para alguna de las palabras de $Lmax$ el mismo no le asigna el valor correspondiente. Sea entonces $C_{ip}$ dicha palabra, tal que $C_{ip} \in Lmax$. Como el algoritmo asigna a $C_{ip}$ el mayor valor de los edit step anteriores mas uno, quiere decir que existe una palabra $C \ne C_{i(p-1)}$ que es edit step de $C_{ip}$ y tiene asignado un valor mas grande que $C_{i(p-1)}$. Luego se puede ver que el valor del edit step ladder de $C_{i1}, C_{i2}, ... , C_{i(p-1)},C_{ip}, ... , C_{ij}$ $=$ $Lmax$ es menor que el generado ahora, que es $C_{i1}, C_{i2}, ...,C, C_{ip} , C_{ij}$, por lo tanto quiere decir que existe un edit step ladder de largo mayor a $Lmax$, pero esto es absurdo, por que $Lmax$ era la sub lista de $L$ de edit step ladder máximo.

Como se puede observar en el ejemplo anterior, con dicho algoritmo se puede llegar a la solución buscada. Hay que tener en cuenta que existen otros algoritmos que también pueden llegar a la solución, por ejemplo hacer, por cada palabra que se recorre, verificar todas las anteriores viendo si existe una que es edit step de la actual, en este caso si bien se llegaría a resolver el problema, el algoritmo no utiliza las propiedades de la lista de palabras ingresada (se encuentran ordenadas alfabéticamente), y esto repercute en el tiempo requerido para encontrar el valor buscado que es mayor. El algoritmo mostrado, tiene en cuenta dicha cualidad y de esta forma es posible bajar el tiempo requerido para hallar el valor buscado.


\subsection{Cálculo de complejidad}

En ésta sección, demostraremos la complejidad computacional de la solución creada. Para calcular dicha complejidad, utilizaremos el pseudocódigo del algoritmo y describiremos cuales son los costos de realizar las operaciones involucradas en cada línea.

El pseudocódigo es el siguiente:

\begin{algorithm}[H]
\caption{Calcula el step ladder de largo máximo}
\label{alg:algoritmo1b}
\begin{algorithmic}[1]
\PARAMS{cantidad de palabras}
\STATE inicializar los maximos step ladders de cada palabra en 1
\STATE inicializar maxLadder = 1
\FOR{cada palabra P}
	\STATE \COMMENT{borrado de letras}
	\STATE borrar de a un caracter por vez de P y buscar esta nueva palabra en las palabras anteriores a P
	\STATE si se encuentra actualizar el max step ladder de P
	\STATE \COMMENT{reemplazo de letras}
	\STATE reemplazar las letras de P de a una por vez por las letra del abecedario desde `a´ hasta la letra a cambiar, formando $P'$
	\STATE buscar $P'$ en las palabras anteriores a P
	\STATE si se encuentra actualizar el max step ladder de P
	\STATE \COMMENT{agregado de letras}
	\STATE agregar letras de a una por vez en cada posicion de P formando $P'$
	\STATE buscar $P'$ en las palabras anteriores a P
	\STATE si se encuentra actualizar el max step ladder de P
	\STATE actualizar maxLadder
\ENDFOR
\RETURN maxLadder
\end{algorithmic}
\end{algorithm}

Como primer paso, definimos algunas cuestiones con respecto respecto a los simbolos que se utilizarán para demostrar la complejidad del algoritmo.

El enunciado menciona que, se recibirá una lista de palabras ordenadas alfabeticamente. Estas palabras son la entrada del problema, y de ésta entrada depende nuestra solución, por este motivo vamos a tomar a la cantidad de palabras como parametro para calcular la complejidad. A dicha cantidad la definiremos como $N$, y de ésta forma se hará referencia a la cantidad de palabras de entrada en los proximos párrafos.

Además se menciona, que el largo de las palabras se encuentra acotada. Si bien dicho valor no va a resultar importante en la complejidad final (ya que la misma se expresará con el valor de una función que crezca asintoticamente igual) lo usaremos en el calculo intermedio y dicho valor lo denominaremos $L$. Si bien el largo de las palabras no son iguales, podemos decir que $L$ se corresponde con el largo de la palabra mas larga dentro de la lista.

Las primeras dos lineas son inicializaciones de variables, donde en la linea 1 se inicializa un arreglo que tiene tantas posiciones como palabras en el arreglo de palabras (que tiene costo lineal con respecto a la cantidad de palabras). La segunda es una asignacion normal a una variable (de costo constante). De las observaciones anteriores podemos decir que hasta la linea 2, existe una complejidad lineal en función de la entrada ($N$).

En la linea 3 comienza un ciclo de tipo $for$ que termina en la linea 16, aquí se recorren todas las palabras de la entrada (se corresponde con el paso número $1$ en la introducción de la solución), dicho ciclo entonces se hará tantas veces como palabras haya en la entrada, es decir $N$ veces.

En las partes comentadas como ´borrado de letras´, ´reemplazo de letras´ y ´agregado de letras´ se efectua el paso número $2$, que consiste en generar los posibles edit step de la palabra actual. Las tres partes son similares en cuanto a pasos que se realizan para obtener los posibles edit step. La Idea principal en las tres partes consiste en recorrer cada posición de la palabra y realizar una operación (ya sea para modificar, agregar o quitar una letra) y luego ver si la nueva palabra obtenida se encuentra en la lista de palabras que están antes de $P$. Recorrer cada posición de una palabra queda en función del largo de la palabra ($L$). Como la lista de palabras se encuentra ordenada alfabeticamente, ver si cada nueva palabra generada está dentro de la lista antes de $P$ se puede saber realizando un busqueda binaria, que queda en función del logaritmo de la cantidad de palabras recorridas ($log_2(K)$, donde $K$ es la cantidad de palabras ya recorridas) y $L$ (en donde se realiza la comparación), por esta razón la busqueda queda en funcion de $L*log_2(K)$. Esta búsqueda se puede realizar en ese tiempo ya que se utiliza el método $binary\_search$ que brinda la $STL$ de $C++$. Resumiendo, hacer cada una de las acciones en una palabra queda en función del largo $L$ (para recorrer cada posicion de $P$) y $L*log_2(K)$, por lo tanto hacer cada una queda en función de $L^2*log_2(K)$ (donde $K \le N$)

Como dijimos en el párrafo anterior. la idea de ´borrado de letras´, ´reemplazo de letras´ y ´agregado de letras´ consiste en recorrer cada una de las posiciones de $P$. Pero en los casos de  ´reemplazo de letras´ y ´agregado de letras´, se agrega un ciclo mas, que consiste en recorrer las letras del abecedario, entonces se agrega una constante más a la complejidad (la constante 26), que a la hora del calculo final será despreciable, pero que a esta altura la pondremos. Entonces, para estas 2 acciones la complejidad queda en $26*L^2*log_2(K)$ cada una.

Con lo explicado anteriormente podemos ver que por cada ciclo del $for$ principal, se realizan 3 modificaciones de la palabra $P$, donde en cada una se recorre posición por posición de $P$ y se busca en las palabras anteriores. Luego podemos decir que hacer un ciclo esta en funcion de hacer 3 veces el recorrido letra por letra y por cada una de ellas hacer una busqueda y comparación, que resultan en un costo de $L^2*log_2(K) + 2*26*L^2*log_2(K)$ (donde $K$ es la cantidad de palabras ya recorrida). Luego esto se realiza $N$ veces, entonces nos queda algo como lo siguiente:

\medskip
\centerline{ $(L^2*log_2(1) + 52*L^2*log_2(1)) + L^2*log_2(2) + 52*L^2*log_2(2) + ... + (L^2*log_2(N) + 52*L^2*log_2(N))$ }

\centerline{=}

\centerline{ $53*L^2*log_2(1) + 53*L^2*log_2(2) + ... + 53*L^2*log_2(N)$ }

Dicha cuenta la podemos acotar superiormente, considerando que cada valor dentro de los logaritmos, son menores o iguales a $N$. Lo que resulta en lo siguiente:

\centerline{ $53*L^2*log_2(1) + 53*L^2*log_2(2) + ... + 53*L^2*log_2(N) {\rm  \ } \le {\rm  \ } 
53*L^2*log_2(N) + 53*L^2*log_2(N) + ... + 53*L^2*log_2(N)$}

\centerline{=}

\centerline{$53*L^2*N*log_2(N)$}

A su vez, $L$ puede ser acotado superiormente por $16$, lo que nos daría que la complejidad estaría en función de $53*16^2*N*log_2(N)$. Teniendo en cuenta el orden asintotico de esta complejidad, podemos despreciar las constantes y finalmente decir, que el algoritmo de solución esta en el orden de:

\medskip
\centerline{$O(N*log_2(N))$}
